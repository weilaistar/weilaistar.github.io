Linux中内存映射与解映射函数

2025-12-01 Linux 内存管理 内存映射

在Linux中，内存映射（mapping）和解映射（unmapping）是管理虚拟内存与物理内存之间关系的关键机制。本文将介绍Linux中常用的内存映射与解映射函数及其工作原理。

用户空间的映射与解映射

1. mmap()

mmap() （memory map）是一个系统调用，用于在调用进程的虚拟地址空间中创建一个新的映射。其作用如下：

  • 提高I/O效率：将一个文件或设备映射到内存，使得程序可以像访问内存一样访问文件内容，无需显示调用read或write函数（无需缓冲区来回拷贝）。
  • 共享内存：允许多个进程共享同一块内存区域，便于进程间通信。
  • 匿名映射：可以创建没有关联文件的内存映射区域，常用于共享内存。
  • 内存保护：可以设置映射区域的保护标志，如只读、读写等，防止程序对映射区域的非法访问，这个与MMU（内存管理单元）的页表机制相关。

当进行mmap()这样一个系统调用时，Linux内核会如何处理？

  • 检查参数：内核会检查参数是否有效，如地址是否对齐、映射长度是否为页大小的整数倍等。
  • 分配内存：根据参数，内核会分配足够的内存页来满足映射请求。内核会创建一个或多个 vm_area_struct (VMA) 结构来描述这段虚拟内存区域，并将其插入到进程的虚拟内存结构中。
  • 更新页表：将新映射的虚拟地址与物理内存页建立映射关系，VMA 会记录与哪个文件关联，更新MMU的页表。（在这个阶段，通常并不会立即建立页表条目<Page Table Entries, PTEs>到物理页的映射，而是采用延迟分配，直到进程实际访问该虚拟地址导致缺页异常时才真正分配物理页并建立映射。）
  • 返回映射地址：将映射的起始地址返回给调用进程。
    对于设备驱动程序，如果它实现了 file_operations 结构中的 mmap 接口，mmap() 可以将设备内存直接映射到用户空间。

其函数原型如下：

  • addr：建议的映射起始地址，通常设为 NULL 让内核选择地址。
  • length：映射的长度。
  • prot：映射区域的保护标志，如 PROT_READ、PROT_WRITE。
  • flags：映射的类型，如 MAP_SHARED 或 MAP_PRIVATE。
  • fd：要映射的文件描述符。
  • offset：文件映射的起始偏移量。

2. munmap()

munmap() 函数用于解除之前通过 mmap() 映射的内存区域。其函数原型如下：

  • addr：要解除映射的地址。
  • length：要解除映射的长度。

内核空间的映射与解映射

1. ioremap()

ioremap() 函数用于将物理内存映射到内核地址空间，是专门用于处理设备 I/O 内存的函数。

为什么要专门为设备 I/O 内存设计 ioremap() 函数？
CPU 在设计上将物理 RAM 和设备 I/O 内存（Memory-Mapped I/O, MMIO）视为不同的资源。这些 I/O 内存区域位于系统总线上，通常不能直接使用常规的内存访问指令（如读取 RAM 的指令）来访问，并且它们具有特殊的内存属性。
因此，Linux 内核提供了 ioremap() 函数来正确地映射这些 I/O 内存区域，使内核能够安全地访问设备寄存器和缓冲区。

其函数原型如下：

  • phys_addr：要映射的物理地址。
  • size：映射的长度。

2. iounmap()

iounmap() 函数用于解除之前通过 ioremap() 映射的内存区域。其函数原型如下：

  • addr：要解除映射的地址。

总结

内存映射与解映射是Linux内存管理的重要组成部分。通过 mmap() 和 munmap()，用户空间程序可以高效地访问文件和设备；而通过 ioremap() 和 iounmap()，内核模块可以安全地访问物理内存区域。理解这些函数的使用，有助于开发高效的应用程序和内核模块。
